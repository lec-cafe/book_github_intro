#GitHub Forkからcheckoutまでの活用
##GitHub Fork

まず GitHub で共同開発を行うことになった時に、一番初めに行わなければならないのが`Fork`という作業です。

`Fork`とは共同で作業しているリモートリポジトリの内容をコピーして自分のリモートリポジトリ上に同じ内容のリポジトリを作成する機能です。

GitHubの右上にある `Fork` ボタンをクリックすると自動的に共同で作業を行っているリポジトリの内容のコピーを自分のリモートリポジトリに作成することができます。
リポジトリの内容がコピーされていることは自分のGitHubのプロフィール画面から確認することができるでしょう。

##GitHub clone
Forkを行って自分のリモートリポジトリにコピーされた内容を追加することができました。

しかし、自分のPC上でファイルを編集するためにはリモートリポジトリ上から自分のPCにファイルをダウンロードする必要があります。

GtHubには`clone`と呼ばれるリモートリポジトリの内容を自分のPCにダウンロードする機能があります。

ターミナルで自分の作業したいディレクトリに移動し、以下のコマンドを実行することでダウンロードすることが可能です。
```terminal
$ git clone {リモートリポジトリのURL}
```
リモートリポジトリのURLはGitHub上の `clone or download` ボタンから取得することができます。
また、リモートリポジトリを clone してくる際にはForkしてきたリポジトリのURLを利用してください。

:::tip
cloneを行うのは初めてリモートリポジトリの内容をダウンロードする時のみです。
2回目以降、更新された内容をダウンロードしたいときは `git pull`を活用しましょう:::
##GitHub リモートの追加
`git clone`を行うことで自分のPC上にファイルを追加することができました。

しかし、自分のPC上で変更した内容をリモートリポジトリに反映したりpull requestを出すためには、リモートリポジトリを登録する必要があります。

まず現在登録されているリモートリポジトリの内容を確認してみましょう。

ターミナルで `git remote -v` を実行することで確認することができます。
cloneしてきた直後ではでは以下のように表示されるでしょう。
```terminal
$ git remote -v

origin {リモートリポジトリのURL} (fetch)
origin {リモートリポジトリのURL} (push)
```
:::tip 一番初めに登録されるリポジトリは自動的に`origin`という名前がつけられます。
:::

最新の情報を `pull` や `fetch` を行うためには共同で作業を行っているリモートリポジトリを登録する必要があります。

共同で作業を行っているリポジトリ名は一般的に`upstream`という名前をつけます。実際に以下のコマンドで登録してみましょう。
```terminal
$ git remote add upstream {共同で作業を行っているリポジトリのURL}
```
登録ができると`git remote -v`を実行することで`upstream`が追加されていることが確認できるでしょう。


##git checkout　ブランチの切り方
リモートを追加が完了したところで実際に作業を行う前に`ブランチを切る`という作業を行う必要がります。

自分に割り振られたissueを行う時に、どこをどのように編集したのかissueごとにブランチを切ることでひと目で分かるようにすることができます。

まず今自分がいるブランチを`git branch` コマンドで確認することができます。
```terminal
$ git branch

* master
```
ブランチを切っていない状態では`* master`と表示されるでしょう。
`*`は自分の今いるブランチを示しています。

では、新しくブランチを切ってみましょう。
ブランチ名は一般的に自分の作業を行うissueの番号を登録します。
```terminal
$ git checkout -b {新しく切るブランチ名}
```
うまくブランチを切ることができると`Switched to a new branch {新しく切るブランチ名}`と表示されます。

ブランチがうまく切れていることを確認するために `git branch` コマンドを実行してみましょう。
うまく切れている場合、以下のように表示されるでしょう。
```terminal
$ git branch

* {新しく切ったブランチ名}
  master
```

:::tip 単に`git checkout {ブランチ名}` と入力すると指定した既存のブランチに移動します。`-b`　コマンドを追加することで新規にブランチを切ることができます。
:::

##GitHub pull request
自分の作業を行った変更内容をマージ(本筋のブランチに統合すること)してもらうためにpull requestを作成します。

pull requestを作成することで自分のリモートリポジトリの変更内容を管理者に伝達することができます。

Php Stormを利用している場合、左下の `version control` タブからコミットすることができます。自分が変更したファイルを選択し、左側のチェックマークを選択します。

`commit message` には変更内容がはっきり分かるようにissueのタイトルなどを入力します。
入力後 `commit` ボタンを選択することで変更内容のコミットが完了します。

その後、自分のリモートリポジトリ(origin)に変更の反映を行う必要があります。

`git push {変更内容を反映したいリポジトリ名} {反映したいブランチ名(作業を行ったブランチ名)}`で変更を反映することができます。

```terminal
$ git push origin {作業を行ったブランチ名}
```

ターミナル上に表示されたURLからGitHub上のページに遷移することができます。

ページに遷移後 `create new pull request` ボタンからpull requestを作成することができます。

遷移したページではpull request を送信するブランチの設定を行ったり、タイトル、コメントを記述することができます。
pull requestのタイトルは`作業を行ったissueのタイトル`、コメントには `ref #{issueの番号}` と記述するのが望ましいでしょう。
